<task type="auto">
  <name>Task 2: Create goal allocation and subscription intelligence computation modules</name>
  <files>src/lib/db/goal-allocation.ts, src/lib/db/subscription-intelligence.ts, src/lib/db/queries.ts</files>
  <action>
1. **Create `src/lib/db/goal-allocation.ts`**:

   Export interface:
   ```typescript
   export interface GoalAllocation {
     goalId: string;
     goalName: string;
     goalEmoji: string;
     amount: number;  // allocated amount from surplus
   }
   ```

   `computeGoalAllocations(userId: string, month: number, year: number): Promise<GoalAllocation[]>`:
   - Fetch all goals for user where `target > saved` (not yet fully funded).
   - Call `computeMonthlySurplus(userId, month, year)` from `./calculations.ts` to get surplus.
   - If surplus <= 0, return empty array.
   - Calculate allocations based on priority weights:
     - Sum all priorities. If sum is 0 (all unweighted), split equally.
     - Otherwise, each goal gets `(priority / totalPriority) * surplus`.
     - Cap each goal's allocation so it doesn't exceed `target - saved` (remaining need).
     - Redistribute any excess from capped goals to uncapped goals proportionally.
   - Return array of `GoalAllocation` objects.

   `upsertGoalAllocations(userId: string, month: number, year: number, allocations: GoalAllocation[]): Promise<void>`:
   - For each allocation, upsert a GoalAllocation record with status "pending" using the `@@unique([userId, goalId, month, year])` constraint.

   `applyGoalAllocations(userId: string, month: number, year: number): Promise<void>`:
   - Find all GoalAllocation records for user/month/year with status "pending".
   - For each, update the corresponding Goal's `saved` field by adding the allocation amount.
   - Set each GoalAllocation's status to "applied".
   - Run in a transaction (`prisma.$transaction`).

2. **Create `src/lib/db/subscription-intelligence.ts`**:

   `detectPriceChanges(userId: string): Promise<Array<{ subscriptionId: string; name: string; oldAmount: number; newAmount: number }>>`:
   - Fetch all active subscriptions for user where `previousAmount` is not null AND `previousAmount !== amount`.
   - Return array of subscriptions with price differences.
   - This is a simple comparison — the `previousAmount` field gets set when a new transaction amount differs from the stored amount (handled in the import/transaction flow, not here).

   `detectUnusedSubscriptions(userId: string): Promise<Array<{ subscriptionId: string; name: string; daysSinceLastTransaction: number }>>`:
   - Fetch all active subscriptions for user.
   - For each, find the most recent transaction matching the subscription name (case-insensitive `contains` match on transaction name) within the user's transactions.
   - If no matching transaction found in the last 60 days (from today), flag as potentially unused.
   - Return array with subscription info and days since last match.
   - Use a single query approach: fetch subscriptions, then batch-query latest matching transaction dates.

   `computeSubscriptionBudgetImpact(userId: string, month: number, year: number): Promise<Map<string, number>>`:
   - Fetch all active subscriptions that have a `categoryId` set.
   - Group by categoryId, sum amounts (normalize annual to monthly by dividing by 12).
   - Return Map of categoryId -> total subscription cost for that category.
   - This is used by the spending display to show how much of a category's budget is pre-committed by subscriptions.

3. **Update `src/lib/db/queries.ts`**:
   - In `getGoals()`: Add `priority` to the returned GoalView mapping: `priority: g.priority`.
   - In `getSubscriptions()`: Add flag computation:
     - Import `detectPriceChanges` and `detectUnusedSubscriptions`.
     - Call both detection functions.
     - Build a `flags` array for each subscription based on whether it appears in either detection result.
     - Add `categoryName` by joining with category if `categoryId` exists.
     - Include `categoryId` in the query (use Prisma `include: { category: true }`).

Import prisma from `./prisma` in both new modules.
