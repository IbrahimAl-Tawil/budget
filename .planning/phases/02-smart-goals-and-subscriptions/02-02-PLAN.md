   - Accept `priority` in request body alongside existing fields.
   - Add `priority: Number(priority) || 0` to the `prisma.goal.create` data.

2. **Update `src/app/api/goals/[id]/route.ts` (PATCH)**:
   - Accept `priority` in request body.
   - Add `...(priority !== undefined && { priority: Number(priority) })` to the update data.

3. **Create `src/app/api/goals/allocations/route.ts`**:

   **GET** — Compute and return allocations for a given month/year:
   - Extract `month` and `year` from URL search params (default to current month/year).
   - Call `computeGoalAllocations(user.id, month, year)`.
   - Also fetch existing GoalAllocation records from DB for that month/year to show status.
   - Return JSON: `{ surplus: number, allocations: GoalAllocationView[] }`.
   - Include the surplus amount so the UI can display total available.

   **POST** — Generate and save pending allocations:
   - Extract `month` and `year` from body.
   - Call `computeGoalAllocations(user.id, month, year)`.
   - Call `upsertGoalAllocations(user.id, month, year, allocations)`.
   - Return the created/updated allocation records.

   **PATCH** — Override a specific allocation amount before applying:
   - Accept `{ allocationId: string, amount: number }` in body.
   - Update the GoalAllocation record's amount and set status to "overridden".
   - Return updated record.

   **PUT** — Apply all pending/overridden allocations for a month:
   - Extract `month` and `year` from body.
   - Call `applyGoalAllocations(user.id, month, year)`.
   - Return `{ success: true, applied: number }` with count of applied allocations.

   All endpoints check auth via `getApiUser(request)`. Return 401 if unauthorized.
