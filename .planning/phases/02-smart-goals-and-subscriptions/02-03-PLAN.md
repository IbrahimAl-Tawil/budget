   - Include category name: update the Prisma query to `include: { category: true }` and map `s.category?.name` to `categoryName`.

3. **Update `getSpendingData()` in `src/lib/db/queries.ts`**:
   - Import `computeSubscriptionBudgetImpact` from `./subscription-intelligence`.
   - Call it with userId, month, year.
   - For each SpendCategory in the result, add a `subscriptionCommitted` amount from the impact map (matching by categoryId).
   - Since `SpendCategory` currently uses category name (not id), we need to build a name-to-id mapping or adjust the approach:
     - PREFERRED: Pass categoryId through the spending computation so we can match. Add `categoryId: string` to `SpendCategory` interface in types.ts (additive, non-breaking).
     - Then match `subscriptionImpact.get(category.categoryId)` to add committed amount.
   - Add `subscriptionCommitted: number` to `SpendCategory` interface (default 0 if no subscriptions linked).

   IMPORTANT: The `getSubscriptions` function currently does NOT include category in the Prisma query. Add `include: { category: true }` to the `prisma.subscription.findMany` call.
